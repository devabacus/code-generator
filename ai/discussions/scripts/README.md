# 📋 discuss.py — CLI для обсуждений

## Использование

### Создать новое обсуждение

```bash
# С вопросом (рекомендуется)
python discuss.py new "Какой фреймворк выбрать для Python?"

# Пустое (только тема)
python discuss.py new --empty "Архитектура проекта"
```

**Результат:** `active/3-какой-фреймворк-выбрать.md`

---

### Посмотреть активные

```bash
python discuss.py list
```

**Вывод:**

```
📋 Активные обсуждения:

  1. проверка нового обсуждения
  2. архитектура проекта
```

---

### Закрыть и архивировать

```bash
python discuss.py close 1
```

**Результат:**

- Статус меняется на ✅ Closed
- Файл перемещается в `archive/1-тема/discussion.md`

---

### Создать продолжение

```bash
python discuss.py continue 2
```

**Результат:** `active/2-тема_p2.md` со ссылкой на Part 1

---

## Формат файлов

- Нумерация: `1-тема.md`, `2-тема.md` (без нулей)
- Продолжения: `_p2`, `_p3`
- Заголовки: `##` User, `###` Agents, `####` подсекции

---

## Примеры workflow

### Быстрый вопрос

```bash
python discuss.py new "Как оптимизировать запросы к БД?"
# → Передай файл агенту → Получи ответ → close
```

### Длинное обсуждение

```bash
python discuss.py new "Архитектура микросервисов"
# ... много ответов ...
python discuss.py continue 3  # Когда файл > 200 строк
# ... продолжение в _p2 ...
python discuss.py close 3     # Архивирует все части
```
