# Model policy проекта (пример)

Референс для `ai/project/docs/model-policy.md`. Привязка capability tiers
(см. `core/docs/workflow.md` → «Политика моделей») к конкретным моделям — проектное решение;
core этот файл не перезаписывает. Названия моделей ниже — пример на 2026-07; при смене
модельного парка правится ТОЛЬКО этот файл проекта.

| Tier | Модель (пример) | Примечание |
| --- | --- | --- |
| `frontier` | Claude Fable 5 | Только главная интерактивная сессия владельца, по явному выбору (`/model`). Субагентам запрещён |
| `standard` | Claude Opus 4.8 | Default для executor/reviewer-субагентов |
| `mechanical` | Claude Sonnet 5 | Поиск, массовые правки, первичное чтение |
| `independent_reviewer` | GPT (актуальный флагман OpenAI) | Другой вендор обязателен; недоступен → fail-closed (TASK-011) |

Правила из core (напоминание): субагентам tier/модель назначается явно; наследование
`frontier` запрещено; ночной оркестратор — скрипт, не модель.
