# 📋 discuss.py — CLI для обсуждений

Каждая дискуссия живёт в своей папке:

```
active/5-cmd-base-role/
  discussion.md       # основная
  discussion_p2.md    # продолжение основной (опционально)
  roadmap.md          # дочерняя: обсуждение roadmap по итогам
  tasks.md            # дочерняя: обсуждение задач по итогам
  roadmap_p2.md       # продолжение дочерней (опционально)
```

---

## Команды

### `new` — новая дискуссия

```bash
python discuss.py new "Какой фреймворк выбрать для Python?"
python discuss.py new --empty "Архитектура проекта"
```

**Результат:** `active/3-какой-фреймворк-выбрать/discussion.md` + промпты для агентов.

---

### `spawn` — дочерняя дискуссия

После того как в родительской заполнены `## Decision` и `## Summary`, можно породить дочернюю дискуссию по смежной теме. Контекст из родителя подставится автоматически.

```bash
python discuss.py spawn 5 roadmap   # как итог #5 ложится в roadmap
python discuss.py spawn 5 tasks     # какие задачи завести
```

**Доступные типы:** лежат в `spawn_prompts/`. Чтобы добавить свой — создай
`spawn_prompts/<тип>.md` с текстом User-вопроса для агентов.

---

### `continue` — продолжение (новая часть)

Когда файл стал слишком большим, создай `_p2`:

```bash
python discuss.py continue 5              # продолжает discussion.md -> discussion_p2.md
python discuss.py continue 5 roadmap      # продолжает roadmap.md -> roadmap_p2.md
```

---

### `close` — закрыть и архивировать

```bash
python discuss.py close 5
```

- Статус во всех файлах папки меняется на ✅ Closed
- Папка целиком переезжает в `archive/5-.../`
- Если в `discussion.md` заполнен `## Decision` — автоматически создаётся ADR

---

### `list` — активные дискуссии

```bash
python discuss.py list
```

Показывает ID, тему и перечень файлов внутри каждой папки.

---

### `migrate` — миграция со старого плоского формата

Если в `active/` или `archive/` лежат файлы `{id}-тема.md` (старый формат), команда разложит их по папкам и переименует в `discussion.md` / `discussion_p2.md`.

```bash
python discuss.py migrate
```

Идемпотентно — уже-правильно-разложенные файлы пропускаются.

---

## Правила

- **Одна дочерняя дискуссия одного типа на родителя.** Если `roadmap.md` уже
  закрыт, но нужно вернуться — используй `continue 5 roadmap` → `roadmap_p2.md`.
- **Нет дат в именах файлов.** Дискуссия — это документ с решением, у него одно
  имя. История итераций — через `_pN`.
- **Родитель должен иметь Decision/Summary** перед spawn, иначе контекст пустой.
