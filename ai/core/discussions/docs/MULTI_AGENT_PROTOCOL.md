# AI Discussion Protocol v2

Протокол для мульти-агентных обсуждений через файлы.

---

## Как вызвать агента

Отправь путь к файлу:

```
active/1-тема.md
```

**Агент должен:** прочитать файл, добавить свой ответ в конец.

---

## Формат файлов

- Нумерация: `1-тема.md`, `2-тема.md` (без префиксов, без нулей)
- Продолжения: `1-тема_p2.md`, `1-тема_p3.md`

---

## Иерархия заголовков

| Уровень | Кто    | Пример                                                         |
| ------- | ------ | -------------------------------------------------------------- |
| `##`    | User   | `## User`                                                      |
| `##`    | Агенты | `## Claude — 2025-12-26 12:00`, `## Gemini — 2025-12-26 12:00` |

**Почему:** В Obsidian `###` сворачивается, User всегда виден.

---

## ⚠️ Формат ответа агента (ОБЯЗАТЕЛЬНО)

> [!warning] КАЖДЫЙ ответ начинай с заголовка!
>
> ```
> ## Claude_1
> 2025-12-26 14:30
> ```
>
> Формат: `## ИмяАгента_НомерОтвета` + дата на следующей строке
> Формат: `## ИмяАгента_1` + дата на следующей строке

## Ответы в чате после выполнения команды "1"

После добавления ответа в файл дискуссии, в чате пишешь **краткое подтверждение со ссылкой**:

**Формат:**

```
Ответил в файле [Псевдоним_НомерОтвета](file:///путь/к/файлу.md#L{номер_строки}) — краткое описание ответа.
```

**Пример:**

```
Ответил в файле [Claude_5](file:///G:/Projects/Flutter/sdandroid/discussions/active/3-инструменты-firebase.md#L450) — сравнение Sentry и Posthog, рекомендации по хостингу.
```

> **Важно:** Используй `#L{номер_строки}` для навигации (VS Code не поддерживает якоря по заголовкам в file:// ссылках).

**Структура ответа:**

```markdown
## Claude_1

2025-12-26 14:30

Текст ответа.

#### Подсекция (если нужно)

Детали...
```

## Правила

1. Пиши на русском
2. Будь кратким и конкретным
3. **Структура:** Используй заголовки (###, ####) для своих тезисов.
4. **Ссылки:** При ответе на конкретный тезис, ссылайся на его заголовок: `[[#Заголовок того тезиса]]` или `[[#^blockid]]`.
5. Не указывай следующего агента

---

## Многочастные дискуссии

Если файл разбит на части (`_p2`, `_p3`):

1. **Прочитай ВСЕ части** перед ответом
2. Отвечай в последней по номеру
3. Ссылайся на `Part N` при цитировании старого

## после получения команды на закрытие (2)

Если секции Decision, Summary еще нет, то добавь и заполни:

## Decision

## Summary

## Approved

Если уже секции Decision, Summary заполнены то ты должен рядом в секции approved написать свой псевдоним дату,время и либо зеленую галочку поставить, либо восклицательный знак и указать замечания
