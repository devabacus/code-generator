Планировать рефакторинг нужно так что каждый файл будет небольшим по размеру (меньше 100 строк).
Внимательно проверять чтобы вся логика присутствовала после рефакторинга
Код после рефакторинга должен быть хорошо тестируемым через тесты
При проведении рефакторинга делаем частями, после выполнения каждой части проверяем.
Старый код не удаляем пока полностью не проверим весь рефакторинг.
После завершения независимой части рефакторинга пишем тест для него.
Перед каждый следующим шагом рефакторинга запускаем тесты.
Ведём лог выполнения в docs/progress.md — записываем каждое действие, статус и результат.
После прохождения каждой фазы предлагаем пользователю сделать git commit с подробным описанием сделанных изменений.
